Somir, o diretor de um parquinho de diversões, recebeu uma lista de eventos programados para a próxima semana.

Esta lista foi exportada de um sistema de gerenciamento antigo e está bastante desorganizada. Somir precisa de sua ajuda para formatar essa lista corretamente, para que possa ser facilmente lida e utilizada.

A lista de eventos é fornecida em um formato de texto bruto, onde cada linha representa um evento. Cada evento está descrito com o nome do evento, seguido da data e hora, mas tudo está em uma única linha e separado por vírgulas.

Além disso, algumas linhas estão vazias e devem ser ignoradas. Exemplo (as datas estão no formato ano-mês-dia): 

Show de Mágica,2024-02-01,20:00
Parada de Fantasias,2024-02-02,16:00
Concerto de Rock,2024-02-03,21:00
Noite de Fogos de Artifício,2024-02-05,22:00

Você deve fazer uma página web com um <textarea> para o Somir colar a lista que ele tem no formato bruto, e um botão “Formatar” que vai mostrar numa <table> a mesma lista (cada evento será uma linha da tabela), mas formatada de acordo com as seguintes regras:

  • Remover linhas vazias.
  • Ordenar os eventos do mais recente para o mais antigo.
  • Para cada evento segundo essas regras de formatação, as colunas da tabela deverão ser:
    • nome do evento, data do evento no formato (dia-mês-ano)
    • dia da semana correspondente ao evento (por extenso: “segunda-feira”, ou “terça-feira”, etc.)
    • horário (“21:30”, “09:45”, etc.)

Na lista do Somir, os horários estão no fuso-horário local, e você exibir os dados na tabela também no fuso horário local.
