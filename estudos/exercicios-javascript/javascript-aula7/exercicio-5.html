Partindo da sua solução ao exercício anterior, vamos adicionar funcionalidades de importação/exportação.

Como sabemos, o LocalStorage é um armazenamento interno do navegador web, onde cada site tem seu espaço de armazenamento separado. Então se o usuário trocar de navegador web, no novo navegador o Local Storage não terá os dados do navegador antigo.

Por isso a aplicação de bloco de notas precisa de uma função que permita ao usuário exportar seus dados de um navegador e importar em outro navegador. Para isso, adicione dois botões na página.

Um deles é o botão “Exportar”. Quando clicado, ele abre uma janela modal contendo, em formato string JSON, um objeto com as seguintes propriedades: 

		{
			text:	..., // texto do textarea
			backgroundColor: ..., // cor do plano de fundo
			fontFamily: ..., // família da fonte
			fontSize: ... // tamanho da	fonte
		}

Daí o usuário pode selecionar com o mouse e copiar (Ctrl-C) esse texto, que é o que será importado em outro navegador. O outro botão é o “Importar”.

Quando clicado, ele abre uma janela modal contendo um formulário com: • Um <input type="text"> onde o usuário pode colar o conteúdo copiado. • Um botão “OK” para efetuar a importação. Ao clicar em “OK”, a página converterá a string fornecida em objeto e fará a importação dos dados (conteúdo do textarea e regras de estilo da página), fechando depois o modal. A partir daí, a página terá importado todos os dados do outro navegador, então o usuário poderá continuar usando a página de onde parou.

Como essa funcionalidade requer que o usuário cole no <input> os dados a serem importados, é sempre possível que o usuário cometa algum equívoco (cole os dados errados, ou faltando alguma parte, etc.). Se os dados não forem JSON válido, saiba que a conversão em objeto (JSON.parse) produz um erro de execução (teste aí: tente executar JSON.parse('{"test": 10'), onde o fecha-chaves } está faltando). Se a importação der erro, a página não pode quebrar ! Ela deve capturar o erro, informar ao usuário que a string não é válida, e manter aberto o modal de importação para que o usuário possa tentar de novo.
